# منصة تحفيظ القرآن والحديث وتعلّم العربية — الخطوة الأولى

هذه النسخة تحتوي على:

- صفحة رئيسية متجاوبة مع الهاتف والحاسوب.
- العنوان القرآني: **وَرَتِّلِ الْقُرْآنَ تَرْتِيلًا**.
- اختيار ست لغات: العربية والإنجليزية والفرنسية والإسبانية والألمانية والإيطالية.
- ترجمة فورية لكل نصوص الصفحة وتغيير اتجاهها تلقائياً بين RTL وLTR.
- حفظ اللغة المختارة في المتصفح.
- ثلاثة أقسام رئيسية واضحة.
- بنية جاهزة لإضافة صفحات القرآن والحديث واللغة العربية في الخطوات التالية.

> أزرار الأقسام تعرض الآن رسالة مؤقتة بدلاً من فتح صفحة ناقصة. سيتم ربطها بالصفحات الداخلية في الخطوة التالية.

---

## 1. هيكل المستودع

الملفات الموجودة الآن موضحة بعلامة ✅، والملفات التي سنضيفها لاحقاً موضحة بعلامة ⏳:

```text
quran-learning-platform/
│
├── index.html                         ✅ الصفحة الرئيسية
├── README.md                          ✅ هذا الدليل
│
├── assets/
│   ├── css/
│   │   └── style.css                  ✅ تصميم الصفحة الرئيسية وكل الشاشات
│   ├── js/
│   │   ├── i18n.js                    ✅ تشغيل اللغات والترجمة
│   │   ├── quran.js                   ⏳ وظائف القرآن والكاراوكي
│   │   ├── hadith.js                  ⏳ قراءة الأحاديث
│   │   └── arabic.js                  ⏳ قراءة دروس العربية
│   ├── images/                        ⏳ الصور المستقبلية
│   └── fonts/                         ⏳ خطوط محلية اختيارية
│
├── data/
│   ├── translations.json              ✅ ترجمات الواجهة باللغات الست
│   ├── hadith.json                    ⏳ نصوص الأحاديث وترجماتها
│   └── arabic.json                    ⏳ دروس العربية وترجماتها
│
└── pages/
    ├── quran.html                     ⏳ صفحة حفظ القرآن
    ├── hadith.html                    ⏳ صفحة الحديث
    └── arabic.html                    ⏳ صفحة تعلم العربية
```

**مهم جداً:** يجب أن يكون `index.html` في المستوى الرئيسي للمستودع، وليس داخل مجلد إضافي.

## 2. وظيفة كل ملف موجود

### `index.html`

الصفحة التي تفتح أولاً. تحتوي على العنوان، واختيار اللغة، وبطاقات الأقسام الثلاثة. لا توجد أكواد ضخمة داخلها؛ التصميم والبرمجة في ملفين منفصلين.

### `assets/css/style.css`

يحتوي على الألوان، والتخطيط، والخطوط، وتصميم الهاتف والحاسوب. أي تعديل بصري سنضعه هنا.

### `assets/js/i18n.js`

يقرأ ملف الترجمات، يترجم العناصر، يغير اتجاه الصفحة، ويحفظ اختيار المستخدم في `localStorage`. صُمم ليُستخدم لاحقاً في كل الصفحات الداخلية أيضاً.

### `data/translations.json`

يحتوي على نصوص الواجهة باللغات الست. المفاتيح متطابقة في جميع اللغات، وقد تم التحقق من صحة صيغة JSON.

## 3. رفع المشروع إلى GitHub خطوة بخطوة

### الطريقة الأسهل من موقع GitHub

1. سجّل الدخول إلى حسابك في [GitHub](https://github.com/).
2. اضغط علامة **+** أعلى الصفحة، ثم اختر **New repository**.
3. اكتب اسماً للمستودع، مثلاً:

   ```text
   quran-learning-platform
   ```

4. اختر **Public** حتى يعمل GitHub Pages مجاناً.
5. اضغط **Create repository**.
6. داخل المستودع الجديد اضغط **Add file** ثم **Upload files**.
7. فك ضغط ملف المشروع في حاسوبك أولاً.
8. افتح المجلد `quran-learning-platform` واسحب **محتوياته** إلى صفحة الرفع:
   - `index.html`
   - `README.md`
   - مجلد `assets`
   - مجلد `data`
9. تأكد قبل الحفظ أن `index.html` ظاهر في جذر المستودع، وليس بهذا الشكل الخاطئ:

   ```text
   quran-learning-platform/quran-learning-platform/index.html
   ```

10. في أسفل الصفحة اكتب مثلاً `Initial platform home page` ثم اضغط **Commit changes**.

### إذا لم يسمح المتصفح برفع المجلدات

أنشئ الملفات يدوياً من **Add file → Create new file**، واكتب مسار الملف كاملاً في خانة الاسم. مثال:

```text
assets/css/style.css
```

عند كتابة `/` سيُنشئ GitHub المجلد تلقائياً. كرر ذلك مع:

```text
assets/js/i18n.js
data/translations.json
```

ثم أنشئ `index.html` في جذر المستودع.

## 4. تفعيل GitHub Pages

بعد رفع الملفات:

1. افتح المستودع.
2. اضغط **Settings**.
3. من القائمة الجانبية اضغط **Pages**.
4. تحت **Build and deployment** اجعل **Source** هو:

   ```text
   Deploy from a branch
   ```

5. اختر الفرع:

   ```text
   main
   ```

6. اختر المجلد:

   ```text
   / (root)
   ```

7. اضغط **Save**.
8. انتظر عادة من دقيقة إلى عدة دقائق ثم حدّث صفحة **Settings → Pages**.

إذا كان اسم المستخدم `hafedd1-netizen` واسم المستودع `quran-learning-platform`، فالرابط المتوقع هو:

```text
https://hafedd1-netizen.github.io/quran-learning-platform/
```

أما إذا كان اسم المستودع نفسه `hafedd1-netizen.github.io`، فالرابط يكون:

```text
https://hafedd1-netizen.github.io/
```

## 5. اختبار اللغات

1. افتح رابط GitHub Pages.
2. اختر لغة من القائمة الموجودة تحت العنوان مباشرة.
3. يجب أن تتغير جميع النصوص فوراً.
4. عند اختيار العربية يصبح الاتجاه من اليمين إلى اليسار.
5. عند اختيار أي لغة أخرى يصبح الاتجاه من اليسار إلى اليمين.
6. أغلق الصفحة وافتحها مجدداً؛ ستبقى اللغة المختارة محفوظة.

إذا لم يظهر آخر تحديث، استخدم تحديثاً قوياً للصفحة:

- Windows/Linux: `Ctrl + F5`
- macOS: `Command + Shift + R`

## 6. ملاحظات مهمة جداً

- لا تختبر الصفحة بالنقر المزدوج على `index.html` فقط؛ بعض المتصفحات تمنع قراءة JSON عند استخدام `file://`. الأفضل اختبارها مباشرة على GitHub Pages.
- لا تغيّر أسماء المجلدات أو حالة الأحرف. `data` تختلف عن `Data`.
- ملف JSON لا يقبل فاصلة زائدة بعد آخر عنصر.
- لا ترفع ملف ZIP نفسه إلى المستودع وتتوقع أن يعمل؛ يجب فك الضغط ورفع الملفات والمجلدات.
- الكود القديم المرفق في الطلب يحتوي بعد نسخه من محرر نص غني على رموز مثل `&lt;` وروابط Markdown داخل `href` وأجزاء JavaScript مشوهة. لا تلصقه فوق هذه الصفحة. سننقله إلى `pages/quran.html` وننظفه في الخطوة التالية مع الحفاظ على القراء والإخفاء وإضافة الكاراوكي.

## 7. الخطوة التالية المقترحة

إنشاء `pages/quran.html` و`assets/js/quran.js`، ثم نقل ميزات النسخة القديمة بصورة صحيحة:

1. قائمة السور والقراء.
2. التلاوة الصوتية.
3. إخفاء الكلمات أو الآيات وإظهارها.
4. التكرار والتسجيل.
5. تظليل الآية الحالية.
6. إضافة مزامنة الكاراوكي على مستوى الكلمة عندما تتوفر بيانات توقيت موثوقة، مع بديل آمن على مستوى الآية.
7. ترجمة كل أدوات صفحة القرآن باللغات الست باستخدام نفس `translations.json`.
